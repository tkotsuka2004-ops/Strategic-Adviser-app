# 事業計画書（草案）：MeetMemo

作成日：2026年7月18日
ステータス：草案（Draft）

## 事業情報（前提）

- **事業名**：MeetMemo
- **解決する課題**：議事録作成の負担（年平均320時間／人）、タスクの担当者・期限が曖昧になることによる漏れ・進捗管理の属人化
- **ターゲット顧客**：定例会議が多く、タスク管理が属人化しがちな中堅・中小企業（従業員50〜999名規模）
- **提供する価値・解決策**：会議録音→文字起こし→議事録→決定事項・タスク抽出→担当者への自動通知までをワンストップで提供するSaaS
- **ビジネスモデル**：1人あたり定額のサブスクリプション課金（タスク通知機能込み）
- **想定競合**：Notta、スマート書記（Otolio）、Fireflies.ai、サポットさん
- **自社の強み・差別化ポイント**：①録音〜タスク通知のワンストップ提供、②タスクを受け取る担当者側の体験に最適化した通知設計、③シンプルな価格設計

## 1. エグゼクティブサマリー

MeetMemoは、会議録音をAIが自動で議事録化し、決定事項・タスクを抽出して担当者に自動通知する法人向けSaaSである。

ビジネスパーソンは議事録作成に**年間平均320時間**を費やし、**67%が負担を感じ、62.3%が残業要因**と回答している一方、AIサポートツールの**利用意向は72.6%あるにもかかわらず実導入率はわずか1.4%**という大きなギャップが存在する（事業検証レポート参照）。既存競合（Notta、スマート書記/Otolio、Fireflies.ai）は議事録・要約・ToDo抽出までを提供するが、「担当者への能動的なタスク通知」までを録音からワンストップで提供する製品は手薄であり、ここにMeetMemoの参入余地がある。

国内AI議事録市場は2025年の約180億円から2028年に約450億円へ拡大が見込まれ（[出典](https://it-trend.jp/minutes_tool/article/942-5127)）、市場自体は追い風にある。初年度はPMF検証を最優先とし、3年目に単年度黒字化を目指す計画とする。

## 2. 課題と解決策

### 課題

| 課題 | 裏付けデータ |
|---|---|
| 議事録作成の時間的負担が大きい | 年間平均320時間、週6.13時間を議事録作成に費やす（[キヤノンMJ調査](https://news.mynavi.jp/techplus/article/20230221-2598330/)） |
| 議事録作成が本業を圧迫する | 67%が負担感、60%が通常業務に支障、62.3%が残業要因と回答（[日本経済新聞](https://www.nikkei.com/article/DGXLRSP617470_Y1A900C2000000/)） |
| タスクが「言った言われた」で漏れる／担当者に伝わらない | 議事録作成者が会議に集中できず発言機会を損なうとの回答が62.2%（同上）。決定事項はテキスト化されても、担当者への能動的な配信・通知までは既存ツールでは弱い |
| AIツールへの潜在ニーズはあるが導入が進んでいない | AIサポートツール利用意向72.6%に対し実導入率1.4%（[ASCII](https://ascii.jp/elem/000/004/125/4125819/)） |

### 解決策

MeetMemoは以下のフローを単一プロダクトで提供する。

1. 会議音声を録音・自動文字起こし（Web会議ツール連携によるボット参加にも対応）
2. AIが議事録・要約を自動生成
3. 決定事項・アクションアイテムを自動抽出し、発言内容から担当者・期限を判定
4. 担当者本人へSlack/Teams/メール等でタスクを自動通知、期限前リマインドも実施

「議事録を作る人」だけでなく「タスクを受け取る人」の体験まで一気通貫で設計する点が、既存の議事録特化ツール・タスク特化ツールとの違いである。

## 4. ビジネスモデル

### 課金体系

- **基本方針**：スマート書記（ライセンス＋従量課金）やFireflies.ai（AIクレジット制）のような複雑な料金体系を避け、「1人あたり定額＋タスク通知込み」のシンプルなSaaSサブスクリプションとする。
- **想定プラン例**：
  - スタンダードプラン：月額1,500円/ユーザー（議事録作成＋タスク自動通知、10ユーザー以上から契約）
  - ビジネスプラン：月額2,500円/ユーザー（Slack/Teams/Notion等の外部連携、管理者向け分析ダッシュボード付き）
- **想定顧客単価（ACV）**：1社あたり平均30ユーザー想定 × 月額1,500〜2,000円 ≒ 年間契約額 約54万〜72万円

### 収益化のポイント

- 顧客が「タスクを受け取る全員」に価値を感じる設計のため、契約単位を議事録作成者個人からチーム全体（部署単位）に広げやすく、アップセル（ユーザー数増加）が狙いやすい。
- 「デジタル化・AI導入補助金2026」（2026年3月30日申請受付開始、補助率1/2〜2/3、[出典](https://www.chusho.meti.go.jp/koukai/hojyokin/kobo/2026/260310001.html)）の対象ツール登録を取得できれば、中小企業側の初期導入コストを実質半減させ、価格訴求力を高められる。

## 5. 競合優位性

| 競合 | 特徴 | MeetMemoとの主な差別化 |
|---|---|---|
| Notta（国内シェア2位） | 多言語対応・話者分離・Notta Brain等機能拡張力が高い | タスクの担当者自動通知は付随機能止まり |
| スマート書記／Otolio（国内シェア1位） | 決定事項・ToDo抽出が主軸、録音〜タスク管理を一画面で提供 | 料金がライセンス＋従量課金で複雑・高額（月額10,000円〜＋AIパック） |
| Fireflies.ai（グローバル大手） | CRM連携に強み、無制限文字起こし | AIクレジット制で隠れコストが発生しやすく、日本語精度・国内サポートが弱い |
| サポットさん（タスク化特化） | 議事録の箇条書きからAIがタスク自動生成、導入800社突破 | 録音・文字起こし機能を持たず、既存議事録の貼り付けが前提 |

**差別化の3軸**（[戦略設計レポート](./戦略設計_MeetMemo_2026-07-18.md)より）：

1. **録音〜タスク通知のワンストップ提供**：複数ツールを跨ぐコピペ・連携設定の手間を排除
2. **タスクを受け取る側の体験への最適化**：通知の分かりやすさ・期限管理・リマインドに投資し、契約単位をチーム全体に広げる
3. **シンプルな価格設計**：価格複雑な競合に対し、1人あたり定額で導入障壁を下げる

## 6. 実行計画・ロードマップ（最初の1年）

| 期間 | マイルストーン |
|---|---|
| 0〜3ヶ月目 | MVP開発（録音→文字起こし→議事録→タスク抽出の基本フロー）。ターゲット顧客10〜20社へのヒアリングで①〜③の差別化仮説とタスク通知機能への支払い意欲（WTP）を検証 |
| 4〜6ヶ月目 | β版リリース。無料トライアルで5〜10社の実顧客に導入し、担当者通知の精度・UX改善を反復。デジタル化・AI導入補助金2026の対象ツール登録を申請 |
| 7〜9ヶ月目 | 正式版リリース、有料課金開始。Slack/Teams/Notion等の主要連携先を優先実装。初期の有料契約10〜15社を獲得 |
| 10〜12ヶ月目 | 営業・マーケティング体制を強化し、有料契約25社・ARR約1,300万円を目標にPMFの確度を検証。シリーズA調達に向けた実績作り |

## 7. 財務計画（初年度〜3年目の収支概算）

前提：B2B SaaSの一般的な健全指標（LTV/CAC比率3以上、CAC回収期間12ヶ月以内、月次チャーンレート3%未満、[出典](https://scalebase.com/blog/sales-strategy/unit-economics-toha)）を目安に、シード期のチーム規模（5〜10名程度）・人件費水準（[出典](https://www.soico.jp/seed-stage/)）を踏まえた保守的な概算。実際の数値は資金調達・採用状況により変動する。

| 項目 | 1年目 | 2年目 | 3年目 |
|---|---|---|---|
| 有料契約数（社） | 25社 | 150社 | 500社 |
| ARR（年換算） | 約1,350万円 | 約8,100万円 | 約2億7,000万円 |
| 売上高（年度計上ベース） | 約700万円 | 約5,000万円 | 約1億8,000万円 |
| 人件費 | 約3,000万円（5名） | 約5,200万円（8名） | 約1億500万円（15名） |
| マーケティング・営業費 | 約300万円 | 約1,000万円 | 約2,500万円 |
| 開発・インフラ・その他運用費 | 約500万円 | 約800万円 | 約1,500万円 |
| 費用合計 | 約3,800万円 | 約7,000万円 | 約1億4,500万円 |
| 営業利益（概算） | 約▲3,100万円 | 約▲2,000万円 | 約＋3,500万円 |

- 1〜2年目はPMF検証・機能開発優先の先行投資フェーズと位置づけ、赤字を許容。
- 3年目に契約数の積み上げ（500社）とアップセル（ユーザー数増加）により単年度黒字化を目指す。
- 必要資金：シードラウンドで3,000万円〜1.5億円程度の調達を想定し（[出典](https://www.soico.jp/seed-stage/)）、1〜2年目の先行投資（開発費・マーケ費・初期人件費）を賄う。

## 総括・今後の検証事項

- ターゲット企業へのヒアリングで、タスク通知機能への支払い意欲（WTP）と想定単価（月額1,500〜2,500円/ユーザー）の妥当性を検証する。
- デジタル化・AI導入補助金2026の対象ツール登録要件を確認し、価格訴求力・CAC低減効果を具体化する。
- AI事業者ガイドライン第1.2版が新たに定義した「自律型AIエージェント」規定への該当有無を確認し、担当者通知機能のガバナンス設計（人間による最終確認プロセス等）を早期に組み込む。
